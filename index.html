Les balises pour bien commencer en HTML

Les balises de type bloc:

  • <head>: fournit des informations générales (métadonnées) sur le document, incluant son titre et des liens ou des définitions vers des scripts et feuilles de style. ( ne pas confondre avec le hearder)
  • <body>: contenu principal du document HTML. Il ne peux y avoir qu'un élément body par document
  • <header>: groupe de contenu introductif ou de contenu aidant à la navigation
  • <footer> : pied de page
  • <section>: section générique d'un document ( commence généralement par un titre)
  • <nav>: section destinée à la navigation dans un document
  • <div> : permet de structurer le code mais n’a pas de signification ( à utiliser en dernier recours !)
  • <aside> : texte indépendant placé à côté d'un élément mais indirectement lié à un autre contenu
  • <article>: contenu autonome dans le document (peut être lu seul)
  • <h1>: titre principal de la page ( 1 seul par page)
  • <hN>: titre de niveau N, permet de hiérarchiser ses titres ( bien respecter l’ordre et ne pas en sauter)
  • <p>: paragraphe de texte
  • <ul>: liste non ordonnée (li élément de cette liste)
  • <ol> :liste ordonée (li élément de cette liste)

les balises de type inline-bloc

  • <img>: une image dans un document
  • <br>: sauter une ligne dans un paragraphe
  • <button>: créer un contrôle interactif ayant la forme d'un bouton
  • <input> :créer un contrôle interactif dans un formulaire web qui permet à l'utilisateur de saisir des données.
  • <a>: créer un lien

Attributs universels

  • id: identifiant unique d’un élément (un seul par site)
  • class : identifiant qui peut être commun à plusieurs éléments
